## Design

### Entry Point

```
ocr-ticket [command] [options]
```

### Subcommands

#### 1. `single` - 处理单张图片

```bash
ocr-ticket single <image> [-o OUTPUT] [--no-preprocess] [--no-mkldnn] [--threads N]
```

Options:
- `image`: 图片路径 (required)
- `-o, --output`: 输出 JSON 路径 (optional, default: stdout)
- `--no-preprocess`: 跳过预处理
- `--no-mkldnn`: 禁用 MKL-DNN 加速
- `--threads N`: CPU 线程数 (default: 8)

#### 2. `batch` - 批量处理

```bash
ocr-ticket batch <input> [-o OUTPUT_DIR] [--workers N] [--no-preprocess] [--no-progress] [--csv]
```

Options:
- `input`: 图片文件夹或 ZIP 文件路径 (required)
- `-o, --output-dir`: 输出目录 (default: output)
- `--workers N`: 并行进程数 (default: CPU count - 1)
- `--no-preprocess`: 跳过预处理
- `--no-progress`: 隐藏进度条
- `--csv`: 同时导出 CSV
- `--log-dir`: 日志目录 (default: logs)
- `--error-dir`: 错误图片目录 (default: errors)

#### 3. `export` - 导出已有结果

```bash
ocr-ticket export <input> [-o OUTPUT_DIR] [-f FORMAT] [--include-raw]
```

Options:
- `input`: 批量处理结果 JSON 文件路径 (required)
- `-o, --output-dir`: 输出目录 (default: export)
- `-f, --format`: 导出格式 [json|csv|both] (default: both)
- `--include-raw`: JSON 中包含原始 OCR 数据
