## Goal

实现进度展示、错误处理与容错机制，确保批量处理稳定可靠。

## Implementation

### 进度展示

- 使用 `tqdm` 显示处理进度条
- 显示格式: `处理中 [████░░░░░░] 10/100 张`
- 实时更新已处理/总数

### 错误处理

- `BatchResult` 数据类: 追踪成功/失败统计
- `add_success()` / `add_failure()` 方法更新状态
- 错误图片自动复制到 `errors/failed_images/` 目录
- 错误信息保存到 `errors/failed_images/{stem}_error.txt`

### 容错机制

- 单张图片崩溃 (`process_single`) 不影响其他图片
- 使用 `try/except` 捕获所有异常
- `future.result()` 获取结果，即使进程崩溃也返回失败状态

### 核心代码

```python
@dataclass
class BatchResult:
    total: int = 0
    success: int = 0
    failed: int = 0
    results: List[Dict] = field(default_factory=list)
    errors: List[Dict] = field(default_factory=list)

    def add_success(self, image_path: str, result: Dict):
        self.success += 1
        self.results.append({"path": image_path, "status": "success", "data": result})

    def add_failure(self, image_path: str, error: str):
        self.failed += 1
        self.errors.append({"path": image_path, "status": "failed", "error": error})
        self.results.append({"path": image_path, "status": "failed", "error": error})
```

- FileHandler: 记录所有 INFO 级别日志
