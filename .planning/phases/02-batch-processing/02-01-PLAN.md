# Plan 02-01: 批量导入（文件夹、ZIP）与多进程架构

**Phase:** 2 - Batch Processing
**Status:** Completed
**Created:** 2026-04-16

## Goal

实现文件夹和 ZIP 压缩包批量导入图片，使用 multiprocessing 多进程并发处理。

## Implementation

### 批量导入架构

- `BatchProcessor.collect_inputs()`: 统一入口，自动识别文件夹或 ZIP 文件
- `BatchProcessor._collect_images_from_folder()`: 递归扫描支持格式 (jpg, png, webp, bmp, tiff, tif)
- `BatchProcessor._collect_images_from_zip()`: 解压 ZIP 并收集图片

### 多进程处理

- `ProcessPoolExecutor` 实现多进程并发
- `max_workers` 默认设置为 `cpu_count() - 1`
- 单张图片处理在独立进程中执行，崩溃不影响其他进程

### 核心代码

```python
class BatchProcessor:
    def collect_inputs(self, input_path: str) -> List[Path]:
        path = Path(input_path)
        if path.is_file():
            if path.suffix.lower() == ".zip":
                images, _ = self._collect_images_from_zip(str(path))
                return images
            elif self._is_image(path):
                return [path]
        elif path.is_dir():
            return self._collect_images_from_folder(str(path))

    def process_batch(self, image_paths: List[Path], ...) -> BatchResult:
```

## Verification

- 文件夹导入: `processor.process("path/to/folder")`
- ZIP 导入: `processor.process("path/to/archive.zip")`
- 多进程: 观察日志确认使用多个 worker

## Decisions

| Decision | Rationale |
|----------|-----------|
| ProcessPoolExecutor 而非 ThreadPoolExecutor | OCR 推理是 CPU-bound，GIL 影响小 |
| cpu_count() - 1 默认 workers | 保留一个核心给系统 |
